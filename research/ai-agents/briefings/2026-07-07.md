# AI Agents Briefing — 2026-07-07

## Bottom line

Five high-signal updates were added. The main pattern since the last run is operational hardening for long-running agents: workflow fan-out controls and telemetry in Claude Code, canonical command execution records and credential-store serialization in OpenAI Codex, context-compaction training for long-horizon coding agents, and plain-text fallback hardening for Hermes Discord prompts.

## New findings

1. [[Claude Code]] v2.1.202 added advisory dynamic workflow sizing plus workflow `run_id`/`name` OpenTelemetry attributes, improving observability for multi-agent background workflows.
2. [[OpenAI Codex]] migrated shell execution to canonical command lifecycle items, a useful state/audit primitive for coding-agent replay and governance.
3. [[OpenAI Codex]] serialized shared [[MCP OAuth Credential Stores]], preventing lost updates when concurrent MCP credential operations touch aggregate File/Secrets stores.
4. [[CompactionRL]] frames context compaction as part of RL for long-horizon agents and reports gains on SWE-bench Verified and Terminal-Bench 2.0.
5. [[Hermes Agent]] now mirrors Discord interactive prompt payloads into plain message content, improving reliability of asynchronous approval/clarification prompts.

## What changed since last run

- [[Agent Memory Systems]] now has at least three high-signal primary sources summarized: [[ContextSniper]], [[Governed Evolving Memory]], and [[CompactionRL]]. The topic was moved to Done.
- The coding-agent evaluation thread remains Active; today’s CompactionRL finding adds benchmark data, but the requested comparison of scope/limitations still needs synthesis.
- New emphasis: canonical event schemas and shared-state concurrency are becoming recurring implementation concerns in agent runtimes.

## Open questions

- Are advisory workflow-size settings enough for unattended cron use, or should scheduled agents require enforced caps?
- Should canonical command items become a common join point for network/file/process/tool attribution?
- Should scheduled-agent harnesses include concurrency tests for shared memory, credential, and tool-state stores?
- How should Markdown-first memories distinguish lossy compaction summaries from confirmed durable facts?
- Should gateway connectors require self-contained plain-text fallbacks for all approval prompts?
